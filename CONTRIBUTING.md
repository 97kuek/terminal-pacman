# コントリビューションガイド

このプロジェクトへの変更は、遊べる状態を壊さずに少しずつ改善する方針で進めます。

## 開発環境

### Windows

MinGW-w64 GCC を使います。

```powershell
powershell -ExecutionPolicy Bypass -File .\build.ps1
```

実行:

```powershell
.\build\terminal-pacman.exe
```

### Linux / macOS

```sh
make
./build/terminal-pacman
```

## 変更前に読むもの

- `SPEC.md`: 現在のゲーム仕様
- `docs/IMPLEMENTATION_PLAN.md`: 実装構成と進捗
- `docs/ROADMAP.md`: ゲーム性改善の優先度
- `AGENTS.md`: AI エージェント向けの作業方針

## コーディング方針

- C99 を基準にする。
- 外部ライブラリは、必要性が明確になるまで追加しない。
- OS 依存処理は `src/platform.h` の API に閉じ込める。
- ゲームルールは `src/game.c` に寄せ、描画処理と混ぜない。
- 描画は `src/render.c` に寄せる。
- まずは ASCII 表示を維持する。
- 関数は短く保ち、状態変更の責務を分ける。

## 変更の進め方

1. 変更したい内容を `SPEC.md` または `docs/ROADMAP.md` と照らし合わせる。
2. 仕様が変わる場合は、実装前にドキュメントを更新する。
3. 実装する。
4. ビルドする。
5. 可能なら実際にプレイして、主要な操作を確認する。
6. 変更内容に合わせて README や計画書を更新する。

## 確認項目

最低限、以下を確認してください。

- Windows でビルドできる。
- 変更が平台固有でない場合、Linux / macOS でもビルドできる設計になっている。
- `WASD` 操作が壊れていない。
- `Q` で終了できる。
- 端末モードが通常終了時に復元される。
- プレイヤーと敵が壁を通り抜けない。

## コミットメッセージ

短く、何を変えたかが分かる英語または日本語にします。

例:

```text
Add power pellet mode
敵の移動ロジックを分離
```

## 優先して歓迎する変更

- ゲーム性が明確に上がる小さな機能
- 純粋なゲームロジックのテスト追加
- Linux / macOS でのビルド確認
- 端末表示のちらつき軽減
- README や仕様書の改善

## 避けたい変更

- ゲーム性と無関係な大規模リファクタリング
- クロスプラットフォーム性を壊す変更
- 仕様書と実装の差分を放置する変更
- 初期方針と矛盾する重い依存ライブラリの追加
