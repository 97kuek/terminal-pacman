# 仕様書

## 目的

Windows / Linux / macOS のターミナル上で動作する、C 言語製のパックマン風ゲームを実装する。

初期版では、見た目の豪華さよりも次の点を優先する。

- 追加ライブラリなしで動くこと
- 操作できる完成したゲームループがあること
- ソースコードが読みやすいこと
- OS ごとの差分が小さく分離されていること

## 対応環境

- Windows
- Linux
- macOS

初期版では外部ライブラリを使わない。端末入力や画面制御に必要な OS 別処理は、小さな平台レイヤーに閉じ込める。

## 表示

描画文字は ASCII グリフのみを使い、色付けに ANSI（SGR 256 色）エスケープを使う。Windows では仮想端末（VT）処理を有効化する。

| 要素 | 文字 |
| --- | --- |
| 壁 | `#` |
| ドット | `.` |
| パワーエサ | `o` |
| 空白 | 半角スペース |
| プレイヤー | `C` |
| 敵 | `G` |
| 食べられる敵 | `g` |
| ボーナスフルーツ | `F` |

各セルは横 2 文字ぶんに拡大して描画し、端末サイズに合わせて中央寄せする。画面には固定サイズのマップと、スコア・ハイスコア・ライフ・残りドット数・状態・目的・凡例・操作に加えて、スタシス・パルスのチャージメーターと散開／追跡ウェーブの状態を表示する。

マップサイズ:

- 幅 45 文字
- 高さ 21 行

描画は各セルを横 2 文字ぶんに拡大し、端末サイズに合わせて中央寄せする。

推奨ターミナルサイズ:

- 横 95 文字以上
- 縦 32 行以上

## 操作

| キー | 操作 |
| --- | --- |
| `W` / 上矢印 | 上へ移動 |
| `A` / 左矢印 | 左へ移動 |
| `S` / 下矢印 | 下へ移動 |
| `D` / 右矢印 | 右へ移動 |
| `Space` | スタシス・パルス（チャージ満タン時に敵を凍結）／メニューで決定 |
| `P` | 一時停止 / 再開 |
| `R` | クリア / ゲームオーバー後にリスタート |
| `Q` | 終了 |

### 起動オプション

`--level N`（開始ステージ）、`--difficulty easy|normal|hard`、`--speed MS`（フレーム間隔）、`--no-sound`、`--mono`（モノクロ表示）、`--help`。`--level` または `--difficulty` を指定すると起動メニューを省略する。

`WASD` 操作は必須。矢印キー操作も初期実装に含める。

## モード

起動時にモードと難易度を選ぶ（`--mode` / `--difficulty` で省略可）。ハイスコアはモード別に `terminal-pacman.score` へ保存する。

- **Classic**: 既存の 3 ステージ。最終ステージクリアでゲームクリア。ライフ 3。
- **Endless**: ローグライト（ライフ 1、1 ミスで終了）。迷路を食べ尽くすたびに新しい迷路を生成し、進行に応じて敵が速くなる。
- **Time Attack**: 制限時間（既定 120 秒）でスコアを最大化。捕まると短い復帰を挟むが時間は止まらない。

Endless / Time Attack の迷路はランタイムで生成する（`src/maze.c`）。

## ゲームルール

- プレイヤーはライフ 3 で開始する（Classic）。
- ドットを食べるとスコアが増える。
- パワーエサ `o` を食べると一定時間だけ敵を食べられる。
- パワーエサ中の敵は `g` で表示する。
- パワーエサ終了前は敵表示を点滅させる。
- すべてのドットを食べるとクリア。
- ステージをクリアすると次のステージへ進む。
- 最終ステージをクリアするとゲームクリア。
- 敵に触れるとライフが 1 減り、プレイヤーと敵が初期位置に戻る。
- パワーエサ中に敵に触れると、敵だけが初期位置に戻り、追加点を得る。
- パワーエサ中に連続で敵を食べると得点が倍増する（200 → 400 → 800 → 1600）。次のパワーエサで連鎖はリセットする。
- ステージ進行に応じてボーナスフルーツ `F` が一定時間だけ出現し、取得すると追加点を得る。
- 得点を得たときは取得地点付近にスコアを点滅表示する。
- ペレットを食べるとスタシス・パルスのチャージが溜まり、満タンで `Space` を押すと一定時間すべての敵を凍結できる（本作独自の要素）。
- 主要なイベント（ペレット／パワーエサ／敵捕食／フルーツ／パルス／ミス／クリア）で効果音を鳴らす。
- プレイヤーは曲がりたい方向を交差点の手前で予約できる（先行入力）。
- 左右の端にワープトンネルがあり、水平方向に画面の反対側へ抜けられる。
- 敵に触れると短い死亡演出を挟んでからライフが減る。
- ステージクリア時に残ライフに応じたボーナスを加算する。
- 難易度（Easy / Normal / Hard）に応じて敵の速度・ウェーブ長・パルスのチャージ量が変わる。起動時に難易度選択メニューを表示する（CLI で省略可）。
- ライフが 0 になるとゲームオーバー。
- 敵は 4 体で、直線追跡型・待ち伏せ型・挟み込み型・臆病型に分かれて移動する（追跡経路は A* で求める。詳細は [docs/AI_DESIGN.md](docs/AI_DESIGN.md)）。
- 敵は周期的に散開（コーナーへ退く）と追跡（プレイヤーへ収束）のウェーブを切り替える。
- 敵は壁を通過できない。
- プレイヤーは壁を通過できない。
- ステージ開始時とミス後には短いカウントダウンを入れる。
- ドット収集が進むほど敵の移動間隔を短くする。
- ハイスコアは `terminal-pacman.score` に保存する。

## ステージ読み込み

ステージは `levels/stage1.txt`, `levels/stage2.txt`, `levels/stage3.txt` から読み込む。

- 幅は 45 文字。
- 高さは 21 行。
- `C` はプレイヤー初期位置。
- `G` は敵初期位置（4 体ぶん）。
- `#`, `.`, `o`, 半角スペースをマップ要素として扱う。
- ステージファイルが読み込めない場合は、内蔵ステージにフォールバックする。
- `levels/*.txt` と内蔵ステージは `tools/gen_levels.py` で生成して同期させる。

## ゲームループ

固定間隔のループで進行する。

1. 入力をノンブロッキングで取得する。
2. プレイヤーの向きを更新する。
3. プレイヤーを移動する。
4. 敵を難易度に応じた間隔で移動する。
5. 衝突を判定する。
6. 画面を描画する。
7. 次の tick まで待機する。

## 平台レイヤー

端末依存の処理は `src/platform.h` の API に隠す。

- 端末モードの初期化
- 端末モードの復元
- キー入力のノンブロッキング取得
- 画面クリア
- ミリ秒単位の待機

Windows では `_kbhit` / `_getch` と Win32 Console API を使う。Linux / macOS では `termios`, `select`, `nanosleep` を使う。

## 初期版でやらないこと

- Unicode 表示
- 複雑なメニュー
- ネットワーク機能

サウンド（効果音）と A* ベースの敵 AI は実装済み。サウンドは外部ライブラリを使わず、Windows は別スレッドの `Beep`、Unix は端末ベル（BEL）で鳴らす。
