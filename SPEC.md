# 仕様書

## 目的

Windows / Linux / macOS のターミナル上で動作する、C 言語製のパックマン風ゲームを実装する。

初期版では、見た目の豪華さよりも次の点を優先する。

- 追加ライブラリなしで動くこと
- 操作できる完成したゲームループがあること
- ソースコードが読みやすいこと
- OS ごとの差分が小さく分離されていること

## 対応環境

- Windows
- Linux
- macOS

初期版では外部ライブラリを使わない。端末入力や画面制御に必要な OS 別処理は、小さな平台レイヤーに閉じ込める。

## 表示

初期版は ASCII 文字のみで描画する。

| 要素 | 文字 |
| --- | --- |
| 壁 | `#` |
| ドット | `.` |
| 空白 | 半角スペース |
| プレイヤー | `C` |
| 敵 | `G` |

画面には固定サイズの広いマップと、スコア・ハイスコア・ライフ・残りドット数・状態・目的・凡例・操作を表示する。

初期の推奨マップサイズ:

- 幅 59 文字
- 高さ 23 行

推奨ターミナルサイズ:

- 横 80 文字以上
- 縦 35 行以上

## 操作

| キー | 操作 |
| --- | --- |
| `W` / 上矢印 | 上へ移動 |
| `A` / 左矢印 | 左へ移動 |
| `S` / 下矢印 | 下へ移動 |
| `D` / 右矢印 | 右へ移動 |
| `P` | 一時停止 / 再開 |
| `R` | クリア / ゲームオーバー後にリスタート |
| `Q` | 終了 |

`WASD` 操作は必須。矢印キー操作も初期実装に含める。

## ゲームルール

- プレイヤーはライフ 3 で開始する。
- ドットを食べるとスコアが増える。
- パワーエサ `o` を食べると一定時間だけ敵を食べられる。
- パワーエサ中の敵は `g` で表示する。
- パワーエサ終了前は敵表示を点滅させる。
- すべてのドットを食べるとクリア。
- ステージをクリアすると次のステージへ進む。
- 最終ステージをクリアするとゲームクリア。
- 敵に触れるとライフが 1 減り、プレイヤーと敵が初期位置に戻る。
- パワーエサ中に敵に触れると、敵だけが初期位置に戻り、追加点を得る。
- ライフが 0 になるとゲームオーバー。
- 敵はランダム型、追跡型、先読み型に分かれて移動する。
- 敵は壁を通過できない。
- プレイヤーは壁を通過できない。
- ステージ開始時とミス後には短いカウントダウンを入れる。
- ドット収集が進むほど敵の移動間隔を短くする。
- ハイスコアは `terminal-pacman.score` に保存する。

## ステージ読み込み

ステージは `levels/stage1.txt`, `levels/stage2.txt`, `levels/stage3.txt` から読み込む。

- 幅は 59 文字。
- 高さは 23 行。
- `C` はプレイヤー初期位置。
- `G` は敵初期位置。
- `#`, `.`, `o`, 半角スペースをマップ要素として扱う。
- ステージファイルが読み込めない場合は、内蔵ステージにフォールバックする。

## ゲームループ

固定間隔のループで進行する。

1. 入力をノンブロッキングで取得する。
2. プレイヤーの向きを更新する。
3. プレイヤーを移動する。
4. 敵を難易度に応じた間隔で移動する。
5. 衝突を判定する。
6. 画面を描画する。
7. 次の tick まで待機する。

## 平台レイヤー

端末依存の処理は `src/platform.h` の API に隠す。

- 端末モードの初期化
- 端末モードの復元
- キー入力のノンブロッキング取得
- 画面クリア
- ミリ秒単位の待機

Windows では `_kbhit` / `_getch` と Win32 Console API を使う。Linux / macOS では `termios`, `select`, `nanosleep` を使う。

## 初期版でやらないこと

- Unicode 表示
- サウンド
- 複雑なメニュー
- 高度な敵 AI
- ネットワーク機能
