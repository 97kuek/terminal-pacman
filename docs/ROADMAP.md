# 改善ロードマップ

この文書では、ゲーム性を向上させるために追加したい機能を優先度順に整理します。

## 基本方針

現在のゲームは「動く最小版」です。次の改善では、単に機能を増やすよりも、プレイヤーが判断する場面を増やすことを重視します。

優先する体験:

- 危険を避けるだけでなく、攻める判断がある。
- 敵の動きに読みどころがある。
- 1 プレイごとの緊張感が徐々に上がる。
- 失敗してもすぐ再挑戦できる。

## 実装済み

- リスタート操作
- パワーエサ
- 敵ごとの移動方針
- ステージ開始前とミス後のカウントダウン
- 難易度上昇

## 優先度 A: 次に入れたい機能

### 1. ハイスコア保存

ローカルファイルに最高スコアを保存する。

期待できる効果:

- 繰り返し遊ぶ理由ができる。

注意点:

- 保存先は OS ごとに違うため、最初はカレントディレクトリの小さなテキストファイルでもよい。
- セーブ失敗でゲームが止まらないようにする。

## 優先度 B: 体験の磨き込み

### 2. マップ読み込み

外部テキストファイルからマップを読み込めるようにする。

期待できる効果:

- ステージ追加がコード変更なしでできる。
- テスト用の小さいマップを作りやすい。

### 3. 表示のちらつき軽減

毎回全画面クリアではなく、カーソル位置を戻して上書きする。

期待できる効果:

- ターミナル上での見た目が安定する。

### 4. 設定可能なゲーム速度

起動オプションや定数で tick 間隔を変えられるようにする。

期待できる効果:

- 端末や環境差に合わせやすい。
- 難易度調整にも使える。

## 優先度 C: 将来的な拡張

### 5. Unicode 表示モード

ASCII 表示を維持したまま、任意で Unicode 表示を選べるようにする。

### 6. サウンド

平台差が大きいため、初期段階では優先度を下げる。

### 7. 複数ステージ

マップ読み込みと難易度調整が安定してから追加する。

## 推奨する次の実装順

1. ハイスコア保存
2. マップ読み込み
3. パワーエサ終了前の点滅表示
4. 表示のちらつき軽減
5. 複数ステージ

この順番なら、既存構造を大きく壊さずにゲーム性を上げられる。
