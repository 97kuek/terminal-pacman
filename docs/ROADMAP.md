# 改善ロードマップ

この文書では、ゲーム性を向上させるために追加したい機能を優先度順に整理します。

## 基本方針

現在のゲームは「動く最小版」です。次の改善では、単に機能を増やすよりも、プレイヤーが判断する場面を増やすことを重視します。

優先する体験:

- 危険を避けるだけでなく、攻める判断がある。
- 敵の動きに読みどころがある。
- 1 プレイごとの緊張感が徐々に上がる。
- 失敗してもすぐ再挑戦できる。

## 実装済み

- リスタート操作
- パワーエサ
- パワーエサ終了前の点滅表示
- 敵ごとの移動方針
- ステージ開始前とミス後のカウントダウン
- 難易度上昇
- ハイスコア保存
- マップ読み込み
- 表示のちらつき軽減
- 複数ステージ
- 全画面向け HUD
- 59 x 23 の広いステージ

## 優先度 A: 次に入れたい機能

### 1. ステージ選択

起動後にプレイするステージを選べるようにする。

期待できる効果:

- 特定ステージの練習がしやすくなる。
- ステージ追加時の確認がしやすい。

## 優先度 B: 体験の磨き込み

### 2. クリアボーナス

残りライフや残りパワー時間に応じて、ステージクリア時のボーナスを加算する。

期待できる効果:

- ミスを減らすプレイに意味が出る。
- ハイスコア狙いの目標が増える。

### 3. 敵 AI の難易度設定

追跡精度、先読み距離、移動速度を難易度ごとに変える。

期待できる効果:

- 初心者向けと高難易度を分けやすくなる。

### 4. 設定可能なゲーム速度

起動オプションや定数で tick 間隔を変えられるようにする。

期待できる効果:

- 端末や環境差に合わせやすい。
- 難易度調整にも使える。

## 優先度 C: 将来的な拡張

### 5. Unicode 表示モード

ASCII 表示を維持したまま、任意で Unicode 表示を選べるようにする。

### 6. サウンド

平台差が大きいため、初期段階では優先度を下げる。

### 7. 複数ステージ

マップ読み込みと難易度調整が安定してから追加する。

## 推奨する次の実装順

1. ステージ選択
2. クリアボーナス
3. 敵 AI の難易度設定
4. Unicode 表示モード
5. 複数ステージ用のステージ追加

この順番なら、既存構造を大きく壊さずにゲーム性を上げられる。
